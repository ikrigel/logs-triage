üìö Help & Guide

üéØ Getting Started

The Production Logs Triage application helps you analyze logs, identify issues, and manage support tickets.

  1. View Logs: Browse production logs from different log sets (1-5) in the Logs section
  2. Filter & Search: Use filters to find specific services, log levels, or keywords
  3. Run Triage: Analyze logs with AI to automatically generate tickets and insights
  4. Manage Tickets: Create, update, and track support tickets
  5. Export Data: Download tickets as JSON for external use

üìä Dashboard

The dashboard shows a quick overview of your system:

  • Total Logs Processed: Count of all logs analyzed
  • Open Tickets: Number of unresolved issues
  • Critical Issues: Count of high-priority tickets
  • Last Updated: Timestamp of last activity

üìã Logs View

Browse and filter production logs:

  • Log Sets 1-5: Different production scenarios with increasing complexity
  • Filter by Service: View logs from specific services
  • Filter by Level: Show only ERROR, WARN, INFO, or DEBUG messages
  • Keyword Search: Find specific text in log messages
  • Pagination: Navigate through large log sets

üé´ Tickets

Create and manage support tickets:

  • Create Manual Tickets: Add custom tickets with title, description, severity, and affected services
  • View Tickets: See all tickets with status badges and priority indicators
  • Update Status: Mark tickets as Open, In Progress, or Closed
  • Add Comments: Collaborate by adding notes to tickets
  • Bulk Operations: Select multiple tickets for batch delete or export
  • Export as JSON: Download selected or all tickets for external use

ü§ñ Run Triage

Use AI to automatically analyze logs and generate insights:

  • Preset Logs: Analyze pre-loaded production scenarios (Sets 1-5)
  • File Upload: Import logs from your machine (.log, .txt, .json, .csv)
  • URL Fetch: Load logs from a remote URL (HTTP/HTTPS)
  • AI Analysis: Get automatic ticket generation and root cause suggestions
  • Chat Interface: See the AI's thinking process in real-time

‚öôÔ∏è Settings

Configure your environment:

  • API Keys: Store secure API keys for AI providers (Gemini, Claude, Perplexity)
  • Model Selection: Choose preferred AI provider and model
  • Provider Status: Check availability and switch between providers
  • Theme Toggle: Switch between light and dark modes

üí° Tips & Best Practices

  • Use filters to focus on specific services or error levels
  • Run triage on manageable log sets for better AI analysis
  • Export tickets regularly for backup and integration with other systems
  • Update ticket statuses to track resolution progress
  • Use appropriate API keys for your region and preferences
  • Check the About page for technology details and support info

‚ùì Frequently Asked Questions

Q: Which AI provider should I use?

A: Gemini is recommended for speed and reliability. Claude offers advanced reasoning. Perplexity is a good fallback.

Q: Can I import my own logs?

A: Yes! Use File Upload for local files or URL Fetch to load from a web server.

Q: Are my API keys secure?

A: Keys are stored in browser localStorage and never sent to our servers. Handle with care.

Q: Can I export my data?

A: Yes! Export tickets as JSON from the Tickets page for backup or integration.

Q: What log levels are supported?

A: ERROR, WARN, INFO, and DEBUG. Logs are classified based on severity.

About This Application

Production Logs Triage Agent is an AI-powered system that automatically investigates production logs, identifies issues, correlates them with system changes, and creates support tickets with developer suggestions.

The application combines advanced log analysis, intelligent search capabilities, and modern AI models to help DevOps and SRE teams quickly diagnose and resolve production issues.
